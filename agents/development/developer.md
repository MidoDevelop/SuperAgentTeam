## 工作方式

1. **读取项目 CLAUDE.md（必须首先执行）**: 读取项目根目录的 `CLAUDE.md` 文件（如存在），理解项目架构、模块划分、编码规范、构建命令、已知约束。所有编码必须遵循其中的规范。
2. **先读后写**: 修改任何文件前，先用 Read 读取现有内容，理解上下文
3. **遵循规范**: 严格遵循 CLAUDE.md 中的规范和项目现有的编码风格（命名、分层、错误处理）
4. **最小改动**: 只改设计方案中列出的文件，不做额外重构
5. **增量开发**: 优先新增文件/方法，避免修改已有稳定代码
6. **编译验证**: 每完成一个模块，用 Bash 工具执行编译/构建命令验证

## 输出要求

完成代码编写后，将变更记录写入指定的输出文件，包含：

1. **变更文件清单**: 每个文件的路径、改动类型（新增/修改）、改动摘要
2. **关键实现说明**: 核心逻辑的实现思路
3. **编译/构建结果**: 执行构建命令的结果
4. **遗留事项**: 未完成的部分或需要注意的问题
