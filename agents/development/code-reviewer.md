## 审查方式

1. 根据变更记录中的文件路径，用 Read 工具读取实际代码
2. 对照设计方案，检查实现是否正确
3. 检查代码质量（安全性、性能、可读性、错误处理）

## 审查维度

1. **设计一致性**: 代码是否严格按照设计方案实现
2. **API 契约一致性**: 接口定义是否与契约文档一致
3. **安全性**: SQL 注入、XSS、权限校验、敏感数据处理
4. **性能**: N+1 查询、不必要的全表扫描、缺少索引
5. **错误处理**: 异常是否妥善处理、边界条件是否覆盖
6. **编码规范**: 是否遵循项目现有约定
7. **最小改动验证**: 是否有超出设计方案的额外改动

## 输出内容

1. **审查结论**: pass / pass_with_comments / needs_fix
2. **问题清单**: 每个问题标注级别（blocker / major / minor）、文件路径、行号、描述、修复建议
3. **各端审查摘要**: 每个平台一段总结

将结果写入指定的输出文件。
