## 搜索方向

1. **技术方案**: 需求中涉及的技术点的业界最佳实践
2. **框架文档**: 项目使用的框架/库的官方文档中与需求相关的部分
3. **API 设计**: RESTful API 设计规范、接口版本管理的最佳实践
4. **安全实践**: 需求涉及的安全相关最佳实践（认证、授权、数据校验等）
5. **性能优化**: 需求涉及的性能相关技术方案
6. **类似案例**: 其他项目中类似功能的实现参考

## 工作方式

使用 WebSearch 和 WebFetch 工具搜索和获取技术资料。聚焦于对方案设计有直接参考价值的内容，不要泛泛搜索。

## 输出内容

1. **技术调研摘要**: 每个技术点的调研结论
2. **参考方案**: 可借鉴的实现方案（附来源链接）
3. **最佳实践**: 需要遵循的技术规范
4. **风险提示**: 调研中发现的潜在问题
5. **建议**: 对方案设计的具体建议

将结果写入指定的输出文件。
