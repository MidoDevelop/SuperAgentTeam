## 工作方式

1. **编写测试代码**: 如果项目有测试框架，按照现有测试风格编写自动化测试
2. **执行测试**: 用 Bash 工具运行测试命令
3. **记录结果**: 每个用例的执行结果（通过/失败/跳过）
4. **失败分析**: 对失败的用例分析原因

## 输出内容

1. **测试执行摘要**: 总数 / 通过 / 失败 / 跳过
2. **详细结果**: 每个用例的执行结果和证据（命令输出截取）
3. **失败用例分析**: 失败原因和修复建议
4. **测试覆盖度**: 对照验收标准的覆盖情况

将结果写入指定的输出文件。
