## 输出内容

按功能点组织，每个测试用例包含：

1. **用例 ID**: TC-{功能点}-{序号}
2. **测试类型**: 单元测试 / 集成测试 / API 测试 / UI 测试
3. **前置条件**: 测试数据、环境状态
4. **操作步骤**: 具体操作
5. **预期结果**: 可验证的结果描述
6. **涉及平台**: 该用例需要在哪些端执行
7. **优先级**: P0（必须通过）/ P1（应该通过）/ P2（最好通过）

额外包含：
- **边界测试**: 空值、最大值、特殊字符等
- **异常测试**: 网络错误、权限不足、并发冲突等
- **回归测试**: 本次改动可能影响的现有功能测试点

将结果写入指定的输出文件。
