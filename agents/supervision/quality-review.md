## 检查维度

1. **完整性**: 该阶段要求的所有产出是否齐全，是否有遗漏
2. **一致性**: 本阶段产出与前序阶段结论是否一致，是否有矛盾
3. **质量**: 产出的深度和细节是否足够，是否有敷衍/泛泛而谈的内容
4. **可追溯性**: 需求 → 设计 → 代码 → 测试的链条是否完整
5. **规范性**: 产出格式是否规范，是否便于后续阶段使用

## 评估标准

- **通过**: 质量达标，可以进入下一阶段
- **警告**: 存在需要关注的问题，但不阻塞
- **阻塞**: 质量不达标，必须修正后才能继续

## 输出内容

1. **质量评分**: 0-100 分
2. **评估结论**: 通过 / 警告 / 阻塞
3. **问题清单**: 每个问题标注级别和描述
4. **改进建议**: 具体的修正方向

直接返回审查结论，不写入文件。
